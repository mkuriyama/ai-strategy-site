*[生成AI]: 文章・画像・音声・プログラムなど、新しいコンテンツを「生成」できるAIの総称。ChatGPTやGemini、Claudeなどが代表例。
*[LLM]: Large Language Model（大規模言語モデル）。大量の文章データから言語のパターンを学習した巨大なAIモデルで、生成AIの中核技術。
*[基礎モデル]: GPT、Gemini、Claudeなど、AIサービスの土台となる大規模モデル（Foundation Model）。
*[プロンプト]: AIに対する指示文・入力文のこと。「何をしてほしいか」をAIに伝える文章。
*[トークン]: AIがテキストを処理する際の最小単位。料金計算や処理量の単位として使われる。
*[コンテキスト窓]: AIが一度に読める情報量の上限。「200Kトークン」などと表記され、大きいほど長い文書を扱える。
*[コンテキストウィンドウ]: AIが一度に読める情報量の上限。「200Kトークン」などと表記され、大きいほど長い文書を扱える。
*[カットオフ日]: AIモデルが学習に使ったデータの最終日。これ以降の出来事はAI自身の知識には含まれない。
*[ハルシネーション]: AIがもっともらしいが事実と異なる情報を生成してしまう現象（幻覚）。
*[マルチモーダル]: テキストだけでなく、画像・音声・動画など複数の情報形式を扱える能力。
*[ファインチューニング]: 汎用的なAIモデルを、特定の業務や領域のデータで追加学習させて精度を高めること。
*[RAG]: Retrieval Augmented Generation（検索拡張生成）。回答時に社内文書やDBから関連情報を検索して参照する仕組み。
*[エージェント]: AIが自ら判断して複数の作業を順番に自律的に実行する仕組み。
*[推論モード]: AIが回答前に「考える過程」を明示的に実行するモード（Thinking）。複雑な問題でより正確な回答が期待できる。
*[チャットAI]: ブラウザやアプリで対話形式でAIを使うサービス。最も手軽な利用形態。
*[ChatGPT]: OpenAIが提供する対話型AIサービス。最も利用者が多く、Canvas機能などを備える。
*[Gemini]: Googleが提供する対話型AIサービス／AIモデル。Google各サービスとの連携が強み。
*[Claude]: Anthropicが提供する対話型AIサービス。長文の読解に強い。
*[Copilot]: Microsoftが提供する対話型AIサービス。Microsoft製品との連携が特徴。
*[Grok]: xAIが提供する対話型AIサービス。X（旧Twitter）と連携し、最新情報の反映が速い。
*[スレッド]: AIとの一連の会話のまとまり。新しいスレッドを始めると、それまでの文脈はリセットされる。
*[Canvas]: ChatGPTやGeminiで、コードや文書をチャットとは別の編集画面で作成・修正できる機能。
*[Deep Research]: AIが自動で複数の情報源を調べ、長い調査レポートを作成する機能。
*[Vibe Coding]: AIに自然言語で「こういうものを作って」と伝えてプログラムを書かせる開発スタイル。
*[API]: Application Programming Interface。プログラムからAIサービスを呼び出すための窓口。
*[Google AI Studio]: GoogleのAI開発者向けツール。Gemini APIキーを無料で取得できる（aistudio.google.com）。
*[APIキー]: API利用者を識別するための秘密の文字列（パスワードのようなもの）。他人に見せてはいけない。
*[従量課金]: 使った分だけ料金が発生する仕組み。AIのAPI利用では処理したトークン数に応じて課金される。
*[RPM]: Requests Per Minute。1分あたりのリクエスト数。APIの利用上限を表す指標。
*[TPM]: Tokens Per Minute。1分あたりのトークン数。APIの利用上限を表す指標。
*[RPD]: Requests Per Day。1日あたりのリクエスト数。APIの利用上限を表す指標。
*[レート制限]: APIの呼び出し頻度に設けられた上限（Rate Limit）。超えると一時的に利用できなくなる。
*[JSON]: データの記述形式のひとつ。AIとプログラムの間でやり取りする情報の「入れ物」として広く使われる。
*[構造化出力]: AIの回答を、決まった項目・形式（JSONなど）で返させる仕組み。プログラムで処理しやすくなる。
*[バッチ処理]: データをまとめて一括処理すること。効率化の手法。
*[Python]: AI・データ分析で最も広く使われるプログラミング言語。読みやすい文法が特徴。
*[Google Colaboratory]: Googleが無料で提供するクラウド上のPython実行環境。ブラウザだけで動く。
*[Colab]: Googleが無料で提供するクラウド上のPython実行環境。ブラウザだけで動き、環境構築が不要。
*[ノートブック]: Colabでコードを書いて保存・実行する作業ファイル。複数のセルで構成される。
*[ライブラリ]: 他の人が作った便利な機能の集まり（パッケージ）。既存のものを使って高度な処理ができる。
*[pip install]: Pythonのライブラリをインストール（導入）するコマンド。
*[RSS]: ニュースサイトなどが記事の更新情報を配信する仕組み。
*[エラーメッセージ]: プログラム実行中に問題が発生した際に表示される情報。AIに貼り付けて修正を依頼する。
*[GAS]: Google Apps Script。Google Workspace上で動くプログラム環境。
*[トリガー]: プログラムを自動的に実行する条件設定。「毎朝7時に実行」などの設定ができる。
*[GitHub]: プログラムのソースコードを保管・共有するサービス。
*[Streamlit]: Pythonで書いたプログラムを、ブラウザで操作できるWebアプリに変換するツール。
*[デプロイ]: 作ったアプリやプログラムを、他の人が使える状態にしてサーバー上に配置すること。
*[PoC]: Proof of Concept（概念実証）。アイデアを小規模に試して実現可能性を検証すること。
*[クラウド]: インターネット経由で利用できるコンピュータ資源（サーバー、ストレージなど）の総称。
*[スコアリング]: データに対して点数をつけること。
*[クラスタリング]: 似たもの同士をグループに分ける分析手法。
*[埋め込み]: 文章や画像を、コンピュータが比較・計算しやすい数値の列に変換すること（Embedding）。
*[可視化]: データをグラフや図に変換して視覚的に理解しやすくすること（Visualization）。
*[散布図]: データを平面上の点として配置した図。データ同士の類似性や傾向を視覚的に把握できる。
*[ダッシュボード]: 複数のグラフや指標を一画面にまとめた表示画面。
*[AIX]: AI Transformation。AIを前提として事業構造や組織を根本から再設計すること。
*[AIO]: AI Optimization。顧客が使うAIアシスタントの中で、自社がどう紹介・推薦されるかを意識した施策。
*[モート]: 競合が容易に追随できない防御壁となる競争優位の源泉（Moat／堀）。
*[UVP]: Unique Value Proposition（独自の価値提案）。自社だけが顧客に提供できる独自の価値。
*[VRIO分析]: Value・Rarity・Imitability・Organization の4観点で自社の経営資源を評価するフレームワーク。
*[AI as a Tool]: AIを「便利な道具」として使う段階。
*[AI as a Strategy]: AIを「事業戦略の前提」として組み込む段階。
